01 / Простыми словами
Кто такой FullStack Developer?
Человек, который умеет работать и с тем, что видит пользователь, и с тем, что происходит на сервере.
Frontend

То, что видит пользователь

Кнопки, формы, страницы, анимации, вёрстка — всё, с чем человек взаимодействует в браузере или приложении. Frontend отвечает за визуал, удобство и скорость отклика интерфейса.

Технологии: HTML, CSS, JavaScript, React, Vue, Svelte

Backend

То, что работает под капотом

Серверы, базы данных, расчёты, авторизация, обработка платежей. Backend — это машина, которая обеспечивает логику и надёжность приложения.

Bridge

Связь между мирами

FullStack-разработчик — мост между интерфейсом и сервером. Понимает, как frontend запрашивает данные, и как backend их обрабатывает и возвращает.

02 / Практика
Рабочие задачи
Чем конкретно занимается FullStack-разработчик каждый день.
01

Разработка интерфейсов

Вёрстка страниц, создание адаптивных компонентов, реализация форм и навигации. Контроль pixel-perfect и cross-browser совместимости.

02

Серверная логика

Проектирование API-эндпоинтов, обработка запросов, авторизация пользователей, бизнес-логика — всё, что происходит на сервере до отправки ответа.

03

Работа с базами данных

Проектирование схем, написание оптимизированных запросов, миграции данных. Выбор между SQL и NoSQL в зависимости от задачи.

04

Интеграция API

Подключение внешних сервисов: платёжные системы, почта, CRM, AI-API. Написание и документирование REST или GraphQL интерфейсов.

05

Оптимизация производительности

Профилирование узких мест, кеширование (Redis), ленивая загрузка, оптимизация SQL-запросов, код-сплиттинг. Цель — быстрая загрузка.

06

Тестирование и отладка

Unit-тесты (Vitest, Jest), интеграционные тесты, e2e (Playwright). Поиск и исправление багов, воспроизведение edge-кейсов.

07

Развёртывание

Деплой на VPS или облако, настройка CI/CD, мониторинг логов и метрик, обновление зависимостей, горячие патчи в продакшене.

04 / Путь
Как студенту начать карьеру
Пошаговый план от нуля до первой позиции.
01

Освой основы веб-разработки

HTML для структуры, CSS для стиля, JavaScript для логики. Не беги к фреймворкам — сначала пойми, как работает браузер, DOM, асинхронность и события. Ресурсы: MDN Web Docs.

02

Создавай pet-проекты

To-do приложение, калькулятор, трекер задач, мини-магазин. Каждый проект — решение реальной проблемы. Пушь на GitHub с нормальными коммитами. 3-5 проектов — хорошее начало.

03

Освой Git и GitHub

Branch, merge, rebase, pull request — это повседневная реальность. Заведи профиль, участвуй в open-source, пиши осмысленные коммиты. Работодатели смотрят на профиль.

04

Собери портфолио

Лендинг с описанием проектов, скриншотами и ссылками на GitHub. Покажи не только «что сделал», но и «почему так». Каждый проект решает конкретную задачу.

05

Изучай современный стек

TypeScript, React, Node.js, PostgreSQL, Docker. Не нужно знать всё — выбери одно направление и углубляйся. Следи за трендами: edge computing, AI-интеграции, server components.

06

Готовься к стажировкам

Решай задачи на LeetCode (средний уровень), изучи основы системного дизайна. На собеседовании проверят понимание основ, а не знание фреймворков.

07

Ищи первую работу

Junior FullStack или Frontend/Backend с возможностью расти. Не гонись за зарплатой — важнее опыт и ментор. Стартапы и малые компании — там быстрее растёшь.

05 / Распорядок
Один день из жизни
Типичный рабочий день FullStack-разработчика в компании.
09:00 - 09:30

Планирование задач

Stand-up или личный обзор доски (Jira/Linear). Определяешь приоритеты на день: что деплоить, что рефакторить, что фиксить.

09:30 - 12:00

Глубокая работа

Самое продуктивное время. Пишешь код: реализуешь компонент, серверную логику или интегрируешь API. Наушники, фокус, никаких сообщений.

12:00 - 13:00

Код-ревью

Проверяешь pull request-ы коллег, даёшь фидбек. Или сам выставляешь PR — описываешь изменения, привязываешь тикет.

13:00 - 14:00

Обед

Перерыв. Выходишь из-за компьютера. Важно для концентрации во второй половине дня.

14:00 - 15:00

Встречи команды

Sprint-планирование, обсуждение архитектуры, sync с дизайнером или продактом. Уточняешь требования, оцениваешь сроки.

15:00 - 17:00

Тестирование и отладка

Пишешь тесты, проверяешь edge-кейсы, фиксишь баги. DevTools, логи, мониторинг. Иногда — «работает, но почему-то нет».

17:00 - 17:30

Деплой и завершение

Мержишь изменения, проверяешь CI/CD, деплоишь в staging. Закрываешь тикеты, обновляешь статусы. Фиксируешь, что осталось на завтра.

06 / Миссия
Почему роль важна для ЭРТХ

Создание цифровых продуктов

FullStack-разработчики — ядро команды, которая превращает идеи в работающие сервисы. От внутренних инструментов до клиентских платформ — они строят то, чем пользуются люди каждый день.

Скорость запуска

FullStack закрывает оба конца — не нужно ждать, пока отдельные команды напишут фронтенд и бэкенд. Один специалист может выдать готовый MVP за дни, а не месяцы.

Решение реальных задач

Автоматизация процессов, аналитические дашборды, интеграция с государственными системами, клиентские порталы — задачи, влияющие на эффективность.

Масштабирование

Когда продукт растёт, FullStack адаптирует архитектуру: оптимизирует базы, вводит кеширование, разбивает монолит на сервисы. Он растёт вместе с продуктом.
